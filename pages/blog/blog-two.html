Building my Portfolio

01/11/2019

This is my first portfolio website and I really wanted it to stand out. I took a quick look on Dribbble and Behance for some initial inspiration and got designing.

At first, I put together a simple layout which included an SVG illustration, some titles and a couple of call-to-action buttons. While this design was simplistic and pleasing to the eye, I felt that it lacked character. It was very plain and I thought that the layout was boring and far too common. I ended up scrapping this idea altogether and began searching for different ways of laying out web pages.

Initial Landing Page Design

During my search I came across Jane Simmons’ YouTube channel ‘Layout Land’. She uploads excellent videos about creating web layouts using various techniques, but my favourite video was ‘CSS Grid Like You Are Jan Tschichold’ (Layout Land, 2018) where she shows how using CSS Grid, it’s possible to create layouts in Jan Tschichold’s style. It was exactly what I was looking for. I find the diagonal and vertical positioning very intriguing and uncommon on the web.

Jen's 'Labs' Homepage (Simmons, 2016)

Design Process

I used Figma for the design process because it’s easy to use, has all the necessary tools for high fidelity prototyping. For me, the best thing about Figma is that it works in the browser and all of the files are saved online. Thanks to this it’s possible to edit your projects on any computer with an internet connection without having to install the program. After many tries, I finally had a design I was satisfied with.

Final Landing Page Design

The most common problem I came across was that the layout would have to change a lot depending on the screen size. This is because these designs require a lot of white space. Tschichold claimed that the blank white space is just as important as the areas with black type (Tschichold, 1928).

The cramped space on mobile devices make it hard to create an effective design with a lot of whitespace, but without a lot of scrolling while staying faithful to the original layout on the desktop version. I think that my final layout is a good compromise because it looks pretty much the same whether it’s on mobile or on desktop.

One of Old Landing Page Designs

This design closely resembles Tschichold’s style. It Includes an arrow pointing at the description about me to catch the viewers attention. It makes the most of the widescreen of a desktop page, with the page navigation and titles on the right side, and the description and social links on the left side. Because of this, it would have to change significantly to work on mobile. I also think that these colours made it look too much like Jen Simmons’ Labs page which is why I ended up scrapping this design.

I strayed a little from the intended style, but I’m happy with the final design. I will definitely try my hand at ‘The New Typography’ style again in the future.

Development

Visual Studio Code is my favourite text editor mainly due to the extension support. ‘Live Server’ automatically refreshes the page when saving the file while ‘Live Sass Compiler’ automatically compiles sass without having to install anything but the extension. This was also my first time using Sass, therefore I didn’t use it to its full potential, but I mainly wanted to use it for nesting to make my code easier to manage and using functions to avoid having to write the colour hex values every time.

I also used ‘BEM’ which is short for Block Element Modifier. This is a naming convention which makes code easier to read, work with and understand (Strukchinsky, Starkov, no date). I really like this naming convention and ever since I adapted it into my code I felt less need to use Sass’s nesting feature. In fact, I think that I could ditch Sass entirely since I could also use vanilla CSS variable functions for the colours.

Final Words

I learnt a lot during this project. The New Typography has been very influential to the way I approach design and layouts and I will definitely try to design a website in this style again in the future. Moreover, my CSS Grid understanding has significantly increased thanks to the way these layouts are built.
